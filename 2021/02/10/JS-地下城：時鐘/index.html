JS 地下城：時鐘

發表於 2021-02-10 更新於 2021-02-11

本篇是 JS 地下城第二篇。這次要用 CSS 刻一個時鐘，並用 Javascript 定時更新時鐘上指針的位置，即時反映現在的時間。

閱讀前注意事項：

  • 本文會將 HTML 標籤和 DOM 元素分別略稱為「標籤」和「元素」，並且視兩者為同義詞交替使用。
  • 由於本次的學習重點其中一項為「使用 CSS 畫一個時鐘」，因此除了 Javacript 之外也會說明時鐘的樣式。
  • 為了精簡程式碼，會使用 CSS 預處理器 Sass 來完成樣式。樣式會以 .scss 的格式寫，其他有用到的功能會在文內簡單解釋。

本文除示意圖和連結外，分成三段：指定技術、思考和程式碼說明。指定技術會標示解題時程式碼內必須包含的技術；思考則描述如何規劃整體程式碼架構；程式碼說明則解釋原始碼如何按照思考的規劃逐步完成主題。

示意圖和成品連結

Codepen: https://codepen.io/wind83410/pen/YzGgvwY

指定技術

  • 使用 Javascript 的 Date 物件讀取現在時刻，並用 window.setInterval() 更新時鐘。
  • 用 CSS 刻一個時鐘。

思考

程式碼結構參考自 Codepen 使用者 Ilia 的純 CSS 時鐘。

首先，由於這次主題的成品會即時變化，做之前可以先想有哪些會隨時間改變哪些不會。

會隨時間變化的只有指針的位置，用 Javascript 來動態更新；其他不會隨時間變化的，如刻度和數字的款式、指針的裝飾等，就用 HTML 和 CSS 寫死。

本文採取的步驟是先把時鐘款式寫好，再處理時間即時顯示。

時鐘根據零件可再進一步細分：

  • 外框和圓形底盤（這兩個東西寫 HTML 時能夠一起處理所以併在一起。）
  • 刻度
  • 時針
  • 分針
  • 秒針

時間即時顯示的話，叫瀏覽器每秒用內嵌的 style 旋轉指針到特定位置就能輕鬆搞定。

程式碼

時鐘

第一步要決定時鐘本體的 DOM 結構。以最外層 <div> 作為時鐘本體，其他的零件以子元素的形式加進去；因為時鐘的顯示零件都沿與鐘面垂直、朝觀看者的方向重疊，會用絕對定位來安排位置，所以盡量讓時鐘本體和時鐘零件兩者間的關係為父子元素，如此一來所有零件的參考座標都會是時鐘本體的左上角，這樣排比較方便。

但刻度和整點數字是用無序列表 <ul> 和 <li> 的組合來表示，實際上代表這兩個東西的是 <li>。在這裡也許有人會困惑，啊這樣多一層就不是最外層的子元素了不是嗎，之所以這樣做是為了可讀性：我打算以 JS 加入 DOM 元素的形式新增刻度和數字，多一層 <ul> 可讓他人意識到這個結構的存在，如果沒這層別人光看 HTML 會以為少了一個零件。

除此之外，這裡沒有另外為鐘面的圓形底盤多寫一個 <div>。這是因為把 div.frame-face 的偽元素畫成圓形擺在時鐘中央就能完成，不需要特地多寫。

在開始解釋各部分的樣式前，先設定時鐘的大小為 400px 正方形，邊長用 Sass 變數 $clock-width 存起來。

接下來會按照思考區塊的說明裡時鐘零件的排列順序，逐項完成樣式。

外框和圓形底盤

  • 外框：用 border-radius 完成圓角，由於四個圓角半徑都相同，只需要一個值。
  • 圓形底盤：產生偽元素 div.frame-face::before，把它變成圓形，最後擺到中間。
    1. 生成的偽元素預設為行內元素 (inline element)，無法設定寬高；在父元素設定 display: flex; 使偽元素變成彈性項目 (flex item)，一來可設定寬高，二來排版方便。
    2. 變成圓形的話，寬高設定成一樣搭配 border-radius: 50%; 即可。
    3. 因為偽元素不是絕對定位，若要擺到中央只要寫上 margin: auto; 就可以了。

刻度

因為刻度的位置有規律，利用這點用 Javascript 迴圈和模板自動生成，就不必浪費時間寫重複的標籤。

規律為每五個刻度為一組，而刻度分成兩種：帶有整點數字的主刻度和沒有的副刻度。前者內部又包含兩個數字以及中間的直線，後者只是純粹的圓形。

    簡要說明一下 transform 的變形機制，並解釋為何「先平移再旋轉」的順序在 CSS 內看起來像反過來。transform 的變形機制，是先定義座標系統確定元素座標之後再用矩陣變換座標，要掌握只需要知道兩點：

    • 座標系統：xy 直角座標系統，原點由 CSS 屬性 transform-origin 定義（預設值為元素正中央），x 軸向右、 y 軸向下為正。
    • 變換順序：由右至左。每個變形函數會分別對應一個矩陣，瀏覽器按照函數排序由左至右連乘矩陣，最後乘上座標一次完成所有變形，所以乍看之下變形順序也是由左至右，但不是。變形是透過座標和矩陣的相乘來完成，變形後產生的新座標再乘上下一個矩陣以完成下一個變形，所以論順序的話應該是由最靠近座標的矩陣起算，也就是由右到左。舉例來說，假設套用三種變形，產生三個矩陣，按順序寫作 ABC，元素內的某點座標 x 經變形完成後的座標為 ABCx；但因為 ABCx = A(B(Cx)) 所以變形的順序是 C -> B -> A。

指針

擺放三根指針

指針只要一占空間，就會推擠到在 div.frame-face 其他手足元素，比如圓形底盤，所以首先要用絕對定位脫離預設排列。再來，為了讓元素能向時鐘的指針那樣轉，transform 的原點要移到指針一端並對齊時鐘中心，而 rotate() 留給 Javascript 更新。以下僅以時針為例，秒針和分針的狀況都是類似的。

以 transform-origin 移動 transform 原點：設定 transform-origin 的一種方式，是同時給兩個值以設定 x 座標和 y 座標。transform-origin 的參考點為元素的左上角，x 軸向右、y 軸向下為正。能用方位關鍵字（top、left、right、bottom 和 center）和百分比，百分比在 x 軸上的 0% 和 100% 分別對應元素的左邊界和右邊界，在 y 軸上的 0% 和 100% 則是分別對應元素的上邊界和下邊界。這裡 transform 原點 x 座標釘在下邊界邊長的一半，y 座標則釘在下邊界上方 4px 位置。

這裡需要知道線性變換和矩陣的關係，以及它如何投影整個二維空間。會英文的話，推薦直接看知名數學油土伯 3Blue1Brown「線性代數本質」系列的第三集：線性變換與矩陣，他以動畫精準具現線性變換如何移動空間內每個位置，輕鬆讓初學者洞悉線性變換的本質；影片有附中文字幕但機翻味略重。下文會照影片的邏輯簡要說明，並著重在二維平面上。

下文將提到的「向量」，是起點為原點 指向特定的終點座標、以箭頭表示的數學物件，有方向和長度兩個屬性。比如向量 指的是從原點指到座標 的箭頭，長度是兩個座標的距離 。因向量僅由終點座標定義，所以空間上任一點座標都有其對應的向量，如向量 對應座標，以下「向量」和「座標」將視作同義詞互相替換。

首先，把線性變換 (linear transformation) 拆成兩個部分來解釋：變換 (transformation) 和線性變換。

變換 (transformation) 其實指的是函數。在線性代數的脈絡裡，專指輸入為向量、輸出為向量的函數，也可以把這個函數看做是「把輸入向量的箭頭拉到輸出向量所指的位置」這個過程。

要完全理解變換的話，得同時看空間上所有的向量透過這個變換對應到哪個向量。若把向量看做是空間的座標，等同於看空間上所有點被函數換到哪個位置。若要看變換過後的空間長怎樣，可以看由鉛錘線和水平線構成的網格邊框上的各點，經轉換後所產生的圖像，如下圖所舉的例子。

而這次的重點，線性變換指的是具有兩個性質的變換：直線上的所有點經變換後的點仍會構成直線，不管是直的橫的還是斜的；原點變換過後仍在原點。若事先將二維平面畫上網格，則線性變換後的網格格線會由兩組平行線彼此交錯來構成，組內平行線的間距都相同，網格多呈平行四邊形。如下圖所示，半透明的網格為變換前，顏色較深的則是變換後，平行線分成棕色和黑色兩組。

線性變換如何移動座標，從線性變換後的網格格線特性（兩組平行線，組內平行線間距相同）可知：只要知道 x 方向單位向量 和 y 方向單位向量 這兩個二維空間的基底向量轉換後的位置之後，就可以計算。
